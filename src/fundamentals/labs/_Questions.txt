Answer the following questions:

1.) Using the command line, how do you:
    a. change directories?
    Answer
    You change directories using the command cd [directory_name]

    b. list the contents of a folder?
    Answer
    You list the contents of a folder using the command ls with optional arguments [-altr]

    c. print the path of the current directory?
    Answer
    You print the path of the current directory using the command pwd

    d. move a file from one location to the another?
    Answer
    mv [path of current file] [path of destination location]

    e. copy a file from one location to another?
    Answer
    cp [current file path location] [destination file path location]

    f. create a file?
    Answer
    Go to file location and issue the command vim [filename].

    g. create a folder?
    Answer
    mkdir [folder_name]

    h. delete a file?
    Answer
    rm [file_name]

    i. delete a folder?
    Answer
    rmdir [name of folder]

    j. print the contents of a file?
    Answer
    Cat [name of file]

    k. edit the contents of a file?
    Answer
    Vim [name of file]

2.) Why do we use git and Github?
Answer
We use git to host, track and save changes made to our code. Github is an online repository used to host, track and save changes made to code files

3.) What is the general flow of git?
Answer
-Make changes to your code files (you want to track and backup those changes)
-We use git add to add those changes to the staging area
-We use git commit to commit those changes to the git record
-we use the git push to push those changes to the repository

4.) What is the purpose of adding a remote to a local git repository?
Answer
We add a remote to a local git repository to help push changes made on our local repository to github, which is a remote repository

7.) What is a code block and how do you create one?
Answer
A code block is a group of two or more statements. You create code blocks by enclosing the group of statements between opening and curly braces

8.) What is the idea of variable scope?
Answer
The idea of variable scope is to localize a variable, protecting it from unauthorized access or modification. Codes outside this scope cannot access the  variables within the scope
